# INNER BOOK 数据日报分析

## 技能描述
这是一个专为“INNER BOOK”项目设计的数据分析能力。它每天自动分析和生成数据报告，并响应用户对日报、周报、月报的主动查询。
核心目标：通过Agent Browser skill ，完成“打开指定页面→输入账号密码登录→浏览目标页面→总结页面核心数据”的完整操作

## 获取数据

### 访问页面
1. 使用 Agent Browser 这个 skill 打开页面，在地址栏中，输入目标页面的完整URL，https://inner-book-0gdweqyu8ab70e46-1251129499.tcloudbaseapp.com/app-2cxbe3on/production/u7lkp0rhqz4，输入完成后按下键盘“Enter”键，等待页面加载完成；
2. 页面加载完成后，如果出现了“风险管理”的提示，没关系，等待 5 秒，当“确定访问”的按钮变为可点击状态，点击“确定访问”

### 登录
如果页面没有登录态，可以通过账号密码登录，详细步骤：
1. 在登录页面，上方是一个二维码， 忽略二维码，点击二维码下方的其他登录方式下的“账号密码登录”按钮（classname=provider-button），切换到账号密码登录模式
2. 在两个输入框中分别填入用户名和密码，第一个输入框填入用户名 [yippee701], 第二个输入框填入密码 [123456]
3. 点击登录，如果页面自动跳转到仪表盘并展示数据，则说明登录成功

### 获取数据
1. 页面第一部分是四个卡片，分别是开启对话数、报告完成数、报告解锁数、注册用户数，这些是截止目前的汇总数据，并非是当日的数据
2. 页面第二部分是标题为“报告数量趋势”的折线图，这个根据横轴的日期，可以真正看出每日的数据，开启对话数、报告完成数、报告解锁数这三个是最重要的指标
3. 页面第三部分是对话轮次的数据，只需要分析表格中，创建日期为今日的数据，总结今日数据的对话轮次分布，并且，如果对话轮次有大于11的，属于异常数据，需要强调

### 输出报告
报告内容包括：
- 今日的对话创建数量
- 今日的报告完成数量
- 今日的报告解锁数量
- 今日的对话轮次分布情况
- 今日的营业额  （营业额为报告完成数量 * 6.9元）

同时，除了日报以外，我可能会主动询问你周报、月报，也需要你按照上述格式输出，只需要调整时间周期即可。
